🐳 Container

Tìm hiểu Docker từ đầu

📅 15/01/2025 ⏱ 7 phút đọc ✍ Sinh viên CNTT

Docker là một nền tảng mã nguồn mở giúp đóng gói, phân phối và chạy ứng dụng trong các container — môi trường độc lập, nhẹ và nhất quán trên mọi máy tính. Nếu bạn từng gặp cảnh "chạy được trên máy tôi mà không chạy được trên máy bạn", Docker chính là giải pháp.

Container vs. Máy ảo (VM)

Container chia sẻ kernel của hệ điều hành host, vì vậy chúng khởi động trong vài giây và tiêu tốn ít RAM hơn VM đến hàng chục lần. Đây là lý do container trở thành tiêu chuẩn triển khai microservices hiện đại.

Các khái niệm cốt lõi

Docker Image

Image là bản thiết kế (blueprint) bất biến chứa mã nguồn, thư viện, biến môi trường và cấu hình cần thiết để chạy ứng dụng. Image được xây dựng từng lớp (layer), giúp tái sử dụng và tải về nhanh hơn.

Docker Container

Container là một instance đang chạy của image. Từ một image bạn có thể tạo ra hàng trăm container độc lập, mỗi container có filesystem và network riêng.

Dockerfile

Dockerfile là tập tin văn bản chứa các lệnh để xây dựng một image tùy chỉnh. Mỗi dòng lệnh tạo ra một layer mới trong image.

# ── Ví dụ Dockerfile cho ứng dụng Node.js ──
FROM node:20-alpine

WORKDIR /app

COPY package*.json ./
RUN npm install --production

COPY . .

EXPOSE 3000
CMD ["node", "server.js"]

Docker Hub & Registry

Docker Hub là kho lưu trữ image công khai lớn nhất, tương tự GitHub nhưng dành cho container image. Bạn có thể pull image từ đây hoặc push image của mình lên.

Các lệnh thường dùng

💡 Hầu hết lệnh Docker đều có dạng: docker <noun> <verb> — ví dụ docker image ls, docker container run.

# Kéo image từ Docker Hub
docker pull nginx:latest

# Chạy container (cổng 8080 → 80 trong container)
docker run -d -p 8080:80 --name my-nginx nginx

# Xem container đang chạy
docker ps

# Xem logs của container
docker logs my-nginx

# Dừng và xóa container
docker stop my-nginx
docker rm my-nginx

# Build image từ Dockerfile hiện tại
docker build -t my-app:1.0 .

Docker Compose

Docker Compose cho phép định nghĩa và chạy nhiều container cùng lúc bằng một file docker-compose.yml. Đây là cách phổ biến để phát triển ứng dụng có nhiều thành phần như web server, database và cache.

services:
  web:
    build: .
    ports:
      - "3000:3000"
    depends_on:
      - db

  db:
    image: postgres:16
    environment:
      POSTGRES_PASSWORD: secret
    volumes:
      - pgdata:/var/lib/postgresql/data

volumes:
  pgdata:

Chỉ cần chạy docker compose up -d để khởi động toàn bộ stack. Đây là lý do Docker Compose không thể thiếu trong môi trường phát triển hiện đại.

Kết luận

Docker đã thay đổi căn bản cách chúng ta phát triển và triển khai phần mềm. Bằng cách container hóa ứng dụng, chúng ta có được sự nhất quán giữa môi trường dev, staging và production — giảm thiểu rủi ro và tăng tốc độ delivery. Đây là kỹ năng nền tảng không thể thiếu trong hành trình DevOps và Cloud Engineering.
